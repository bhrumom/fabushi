# 缓存管理功能说明

## 概述

缓存管理功能旨在优化素材下载和存储，避免重复下载，提高发送效率。系统采用双层缓存架构，确保素材的高效复用。

## 缓存架构

### 1. 双层缓存设计

```
用户请求素材
    ↓
检查内存缓存 (Map)
    ↓ (未命中)
检查持久化缓存 (Cache API)
    ↓ (未命中)
从网络下载
    ↓
同时保存到两层缓存
```

### 2. 缓存层级详解

#### 内存缓存 (Memory Cache)
- **存储位置**: Service Worker 内存中的 Map 对象
- **访问速度**: 最快 (微秒级)
- **生命周期**: 浏览器关闭或 Service Worker 重启后丢失
- **容量限制**: 受浏览器内存限制
- **适用场景**: 当前会话中的频繁访问

#### 持久化缓存 (Persistent Cache)
- **存储位置**: 浏览器的 Cache API 存储
- **访问速度**: 快 (毫秒级)
- **生命周期**: 持久保存，直到手动清理
- **容量限制**: 受浏览器存储配额限制
- **适用场景**: 跨会话复用，长期存储

## 缓存策略

### 1. 缓存键生成
```javascript
const cacheKey = `${item.type}_${item.path}`;
// 例如: "r2-asset_佛說大如意寶珠輪牛王守護神咒.amitabha"
```

### 2. 缓存查找顺序
1. **内存缓存优先**: 最快访问，适合当前会话
2. **持久化缓存备用**: 跨会话复用，避免重复下载
3. **网络下载兜底**: 缓存未命中时的最后选择

### 3. 缓存更新策略
- **写入策略**: 下载成功后同时写入两层缓存
- **一致性保证**: 内存缓存和持久化缓存保持同步
- **失效处理**: 下载失败时不更新缓存

## 缓存管理界面

### 1. 统计信息
- **总缓存项目**: 所有缓存中的文件数量
- **总占用空间**: 缓存占用的存储空间
- **内存缓存**: 当前内存中的缓存项数量
- **持久化缓存**: 持久化存储中的缓存项数量

### 2. 操作功能

#### 刷新缓存信息
- 扫描所有缓存存储
- 统计缓存项目和大小
- 更新界面显示

#### 分析缓存使用
- **按类型分析**: 区分不同类型的缓存项
- **按大小分析**: 小文件(<1MB)、中等文件(1-10MB)、大文件(>10MB)
- **按时间分析**: 最近(1天内)、较旧(1周内)、很旧(>1周)

#### 清理过期缓存
- 自动识别超过一周的缓存项
- 批量删除过期内容
- 释放存储空间

#### 清空所有缓存
- 清理内存缓存 (通过 Service Worker)
- 删除所有持久化缓存
- 完全重置缓存状态

### 3. 缓存详情列表
- **文件名称**: 缓存项的原始文件名
- **文件大小**: 占用的存储空间
- **缓存类型**: 内存/持久化/双重
- **缓存时间**: 文件被缓存的时间
- **操作按钮**: 单独删除特定缓存项

## 技术实现

### 1. Service Worker 集成
```javascript
// 缓存检查
if (assetCache.has(cacheKey)) {
    return assetCache.get(cacheKey); // 内存缓存命中
}

const cachedResponse = await cache.match(cacheKey);
if (cachedResponse) {
    const cachedBlob = await cachedResponse.blob();
    assetCache.set(cacheKey, cachedBlob); // 同步到内存缓存
    return cachedBlob; // 持久化缓存命中
}
```

### 2. 缓存写入
```javascript
// 同时写入两层缓存
assetCache.set(cacheKey, blob); // 内存缓存

const response = new Response(blob, {
    headers: {
        'Content-Type': blob.type,
        'Content-Length': blob.size,
        'X-Cache-Time': Date.now() // 缓存时间戳
    }
});
await cache.put(cacheKey, response); // 持久化缓存
```

### 3. 缓存清理
```javascript
// 清理内存缓存
assetCache.clear();

// 清理持久化缓存
const cacheNames = await caches.keys();
await Promise.all(cacheNames.map(name => caches.delete(name)));
```

## 最佳实践

### 1. 缓存策略
- **频繁使用的素材**: 优先保持在内存缓存中
- **大文件**: 主要依赖持久化缓存
- **临时文件**: 设置合理的过期时间

### 2. 存储管理
- **定期清理**: 建议每周清理一次过期缓存
- **空间监控**: 关注缓存占用的存储空间
- **选择性清理**: 根据使用频率决定保留策略

### 3. 性能优化
- **预加载**: 对常用素材进行预缓存
- **批量操作**: 避免频繁的单个缓存操作
- **异步处理**: 缓存操作不阻塞主要功能

## 故障排除

### 1. 缓存不生效
- 检查 Service Worker 是否正常运行
- 确认缓存键生成逻辑正确
- 验证 Cache API 权限

### 2. 存储空间不足
- 使用缓存管理器清理过期内容
- 检查浏览器存储配额设置
- 考虑减少缓存项目数量

### 3. 缓存不一致
- 清空所有缓存重新开始
- 检查缓存更新逻辑
- 验证缓存键的唯一性

## 监控和维护

### 1. 定期检查
