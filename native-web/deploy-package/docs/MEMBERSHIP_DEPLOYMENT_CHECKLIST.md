# 会员系统修复部署检查清单

## 部署前检查

### 1. 代码修复确认
- [x] 统一会员有效期字段名为 `membershipExpiresAt`
- [x] 添加购买记录功能
- [x] 添加兑换记录功能  
- [x] 更新会员中心界面
- [x] 添加新的API接口

### 2. 环境变量检查
确保以下环境变量已正确配置：
- `USERS_KV` - 用户数据存储
- `ORDERS_KV` - 订单数据存储  
- `REDEEM_CODES_KV` - 兑换码存储
- 支付宝相关配置
- Stripe相关配置（如使用）

### 3. KV存储结构
新增的存储键名格式：
- `purchases:{username}` - 用户购买记录
- `redeems:{username}` - 用户兑换记录

## 回滚方案

如果出现问题，可以：

1. **快速回滚代码**：
   - 恢复 `worker.js` 到修复前版本
   - 恢复 `public/membership.html` 到修复前版本
   - 恢复 `stripe-config.js` 到修复前版本

2. **数据安全**：
   - 新增的记录数据不会影响现有功能
   - 可以安全删除 `purchases:*` 和 `redeems:*` 键

3. **API兼容**：
   - 新增的API接口可以安全移除
   - 不影响现有API功能

## 监控要点

部署后需要监控：
- 支付成功率
- 会员激活成功率  
- API响应时间
- 错误日志
- 用户反馈

## 常见问题排查

### 会员状态显示异常
1. 检查 `membershipExpiresAt` 字段是否存在
2. 检查时间格式是否正确
3. 检查时区设置

### 记录不显示
1. 检查API接口是否正常响应
2. 检查KV存储权限
3. 检查前端JavaScript错误

### 支付后会员未激活
1. 检查支付回调处理
2. 检查订单状态更新
3. 检查用户数据更新
