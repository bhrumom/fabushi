# 🔔 锁屏通知栏实时更新功能

## 📱 功能概述

全球法布施系统现已支持锁屏后通知栏实时更新发送进度，让您即使在锁屏状态下也能随时了解发送进度，同时有效防止程序被系统灭活。

## ✨ 主要特性

### 1. 🔒 锁屏状态支持
- ✅ 支持锁屏后继续显示发送进度
- ✅ 通知栏实时更新进度百分比
- ✅ 显示当前正在发送的文件名
- ✅ 显示已完成/总文件数

### 2. 🛡️ 防程序灭活
- ✅ Service Worker 后台持续运行
- ✅ 通知系统保持应用活跃状态
- ✅ 有效防止系统自动清理进程

### 3. 🎯 智能通知管理
- ✅ 自动请求通知权限
- ✅ 静默更新（不发出声音）
- ✅ 点击通知返回应用
- ✅ 支持通知内操作（查看进度、停止发送）

### 4. 🔄 循环模式增强
- ✅ 循环间隔期间显示倒计时
- ✅ 每轮完成后通知提醒
- ✅ 长期运行状态监控

## 🚀 使用方法

### 步骤 1：启用通知权限
1. 首次访问时，系统会自动请求通知权限
2. 在浏览器弹窗中点击"允许"
3. 如果之前拒绝过，可在浏览器设置中手动开启

### 步骤 2：开始发送
1. 正常选择文件并配置发送参数
2. 点击"🚀 开始传送"按钮
3. 系统会显示开始发送的通知

### 步骤 3：锁屏监控
1. 🔒 锁定设备屏幕或切换到其他应用
2. 在通知栏查看实时进度更新
3. 点击通知可以快速返回应用

### 步骤 4：进度交互
- 👁️ 点击"查看进度"返回应用页面
- ⏹️ 点击"停止发送"直接停止任务
- 📱 点击通知主体也可返回应用

## 🔧 技术实现

### 架构设计
```
[主页面] ←→ [Service Worker] ←→ [通知系统]
    ↓           ↓                    ↓
[用户界面]   [后台处理]         [锁屏显示]
```

### 通知更新策略
- **频率控制**：每2秒最多更新一次通知
- **智能合并**：相同文件的多次更新会合并显示
- **电池优化**：使用静默通知减少电量消耗
- **网络友好**：后台任务智能调节网络请求频率

### 兼容性支持
- ✅ Chrome/Edge 88+
- ✅ Firefox 78+
- ✅ Safari 14+
- ✅ 移动端浏览器
- ❌ IE（不支持Service Worker）

## 🔧 自定义设置

### 通知权限管理
可以在浏览器设置中管理通知权限：

**Chrome/Edge：**
1. 设置 → 隐私和安全 → 网站设置 → 通知
2. 找到您的网站，设置为"允许"

**Firefox：**
1. 设置 → 隐私与安全 → 权限 → 通知
2. 找到您的网站，选择"允许"

**Safari：**
1. Safari → 偏好设置 → 网站 → 通知
2. 选择您的网站，设置为"允许"

### 移动端设置
**Android Chrome：**
1. 设置 → 网站设置 → 通知
2. 确保"网站可以请求发送通知"已开启

**iOS Safari：**
1. 设置 → Safari → 通知
2. 允许网站请求通知权限

## ⚠️ 注意事项

### 权限要求
- 必须获得通知权限才能使用此功能
- 部分企业防火墙可能阻止通知功能
- 无痕浏览模式下可能需要重新授权

### 系统限制
- 某些省电模式可能影响通知更新
- 不同操作系统的通知样式略有差异
- 后台应用限制可能影响更新频率

### 最佳实践
- 建议在发送前测试通知功能
- 长时间发送时建议保持设备充电
- 如遇问题可刷新页面重新初始化

## 🆘 故障排除

### 常见问题

**Q: 看不到通知**
A: 检查浏览器通知权限，确保已设置为"允许"

**Q: 通知不更新**
A: 刷新页面重新初始化Service Worker

**Q: 点击通知无反应**
A: 检查是否有多个标签页打开，关闭重复页面

**Q: 移动端通知不显示**
A: 检查系统级通知设置，确保浏览器通知已开启

### 调试工具
1. 打开浏览器开发者工具
2. 查看Console标签的日志信息
3. 检查Application → Service Workers状态
4. 测试页面提供详细的调试信息
