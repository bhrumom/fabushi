# 虚空传输功能

## 概述

虚空传输是全球法布施应用的一项特殊功能，它能够确保数据被真实发送到网络中，即使没有接收设备也能完成传输。这项功能利用了多种网络技术，确保数据包实际离开设备并进入网络。

## 工作原理

虚空传输服务通过以下方式实现真实数据发送：

1. **多端点发送**：同时向多个网络端点发送数据，包括公共DNS服务器、STUN服务器、WebSocket服务器等。

2. **UDP数据包**：在非Web平台上，使用UDP协议发送数据包，这些数据包会真实地进入网络。

3. **WebRTC连接**：在Web平台上，利用WebRTC技术创建点对点连接，即使没有对等方，也能发送ICE候选项和数据包。

4. **WebSocket和HTTP请求**：通过WebSocket连接和HTTP POST请求发送数据，确保数据真实离开设备。

5. **多播地址**：在支持的平台上，向多播地址发送数据，如mDNS(224.0.0.1:5353)和SSDP(239.255.255.250:1900)。

## 使用方法

1. 在应用主界面中，找到"传输模式"卡片。

2. 打开"虚空传输"开关。

3. 选择要发送的文件，然后点击"发送文件"按钮。

4. 观察虚空传输可视化组件，它会显示数据包如何被发送到"虚空"中。

5. 传输统计区域会显示通过虚空传输发送的文件数量和数据量。

## 技术细节

### 非Web平台实现

在非Web平台(如iOS、Android、macOS、Windows)上，虚空传输服务使用以下技术：

- 原生UDP套接字发送数据包
- 多播地址发送
- DNS查询数据包
- ICMP回显请求

### Web平台实现

在Web平台上，由于浏览器安全限制，虚空传输服务使用以下技术：

- WebRTC数据通道和ICE候选项
- WebSocket连接
- HTTP POST请求
- WebTransport(当浏览器支持时)

## 可视化组件

虚空传输可视化组件提供了一个直观的界面，展示数据包如何被发送到"虚空"中。它包括：

- 中心发射点：表示数据的来源
- 飞行的数据包：表示正在传输的数据
- 网格背景：表示网络空间
- 状态指示器：显示当前传输状态
- 传输统计：显示已发送的文件数和数据量

## 注意事项

1. 虚空传输会消耗网络带宽，但不会对网络造成过大负担，因为它使用了控制发送速率的机制。

2. 在Web平台上，某些浏览器可能会限制后台标签页的网络活动，建议保持应用标签页处于活动状态。

3. 虚空传输与其他传输模式(全球发送、WiFi广播)可以同时使用，以确保数据通过多种渠道发送。

## 技术原理

虚空传输的核心原理是确保数据包真实离开设备并进入网络，即使没有明确的接收方。这是通过以下方式实现的：

1. **利用网络协议的特性**：许多网络协议在设计上允许发送数据而不需要确认接收，如UDP。

2. **利用网络服务的回显功能**：某些网络服务(如DNS服务器、STUN服务器)会对特定请求做出响应，这可以用来确认数据已经成功发送。

3. **利用多播和广播**：这些技术允许向多个潜在接收者发送数据，即使没有设备实际接收。

4. **Web平台特殊处理**：在Web平台上，由于直接访问网络的限制，使用WebRTC、WebSocket等技术来确保数据真实发送。

通过这些技术的组合，虚空传输功能能够确保数据真实地进入网络，完成"发送"的目的，即使没有明确的接收设备。
