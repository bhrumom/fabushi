# 全球法布施 - 无中继服务器演示指南

## 快速开始

本指南将帮助您快速体验全球法布施应用中的无中继服务器文件传输功能。这项创新技术允许网页应用直接调用设备功能进行文件传输，无需安装任何额外软件。

### 步骤1：启动应用

1. 打开终端，进入项目根目录
2. 执行以下命令启动Web应用：

```bash
flutter run -d chrome --web-renderer html
```

3. 应用将在Chrome浏览器中打开

### 步骤2：访问P2P演示页面

1. 在应用主界面，点击右上角的"P2P网络演示"图标（WiFi图标）
2. 或者点击主界面下方的"P2P网络演示"卡片
3. 您将进入P2P网络演示页面

### 步骤3：测试点对点连接

1. 在另一个浏览器窗口或标签页中打开同样的应用
2. 两个应用实例会自动发现并连接
3. 在"P2P网络状态"卡片中，您可以看到已连接节点数增加

### 步骤4：传输文件

1. 在任一应用实例中，点击"选择文件"按钮
2. 选择一个或多个文件
3. 点击"广播文件"按钮
4. 文件将直接发送到所有连接的节点
5. 在"传输统计"卡片中，您可以看到传输结果

## 功能演示

### 1. 本地发现演示

要演示本地节点发现功能：

1. 打开多个浏览器窗口或标签页
2. 在每个窗口中打开应用的P2P演示页面
3. 观察每个实例的"已连接节点数"如何增加
4. 查看"已连接节点"列表中显示的节点ID

### 2. 文件传输演示

要演示点对点文件传输功能：

1. 准备一些不同大小的测试文件（小文件、中等文件和大文件）
2. 在一个应用实例中选择这些文件
3. 点击"广播文件"按钮
4. 观察文件如何直接传输到所有连接的节点
5. 比较不同大小文件的传输速度

### 3. 离线工作演示

要演示离线工作功能：

1. 启动应用并进入P2P演示页面
2. 断开互联网连接（关闭WiFi或拔掉网线）
3. 在另一个浏览器窗口中打开应用
4. 验证两个实例仍然可以发现并连接
5. 测试文件传输功能是否正常工作

## 技术演示要点

在演示过程中，可以强调以下技术要点：

1. **无中继服务器** - 整个过程中没有使用任何中继服务器，所有通信都是直接在设备之间进行的

2. **浏览器内信令** - 使用BroadcastChannel和localStorage作为本地信令通道，无需外部信令服务器

3. **自动发现** - 节点自动发现并建立连接，无需用户手动配置

4. **直接传输** - 文件直接从源设备传输到目标设备，不经过任何中间服务器

5. **离线工作** - 在没有互联网连接的情况下也能正常工作

## 常见问题解答

### Q: 为什么需要在同一网络下测试？

A: 当前版本的本地发现机制主要适用于同一局域网内的设备。未来版本将支持跨网络发现和连接。

### Q: 为什么某些浏览器无法使用此功能？

A: WebRTC和其他Web API在不同浏览器中的支持程度不同。Chrome和Firefox通常提供最佳支持。Safari在某些版本中可能需要手动启用WebRTC功能。

### Q: 文件大小有限制吗？

A: 理论上没有硬性限制，但非常大的文件（如超过1GB）可能会受到浏览器内存限制的影响。我们建议对大文件进行分块传输，这在当前实现中已经支持。

### Q: 传输速度受什么因素影响？

A: 传输速度主要受网络带宽、设备性能和浏览器实现的影响。在局域网环境中，通常可以达到接近网络理论速度的传输率。
