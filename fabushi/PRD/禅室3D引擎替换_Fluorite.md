# PRD: 禅室 3D 引擎替换 (高可用方案替代Fluorite)

## 1. 需求背景与目标
**背景：**
目前禅室功能 (`MeditationRoomScreen` 及 `BuddhaModelScreen`) 使用了 `three_js` 的 Flutter 移植版进行 3D 场景渲染。然而考虑到性能瓶颈和设备的兼容性，需要更轻量、更高性能的底层引擎。
鉴于原定目标丰田 `Fluorite` 引擎目前**尚未实际向公众释放 SDK**（无真实可用的仓库/包），本项目转为采用目前行业中最成熟且具备同样（甚至更优）底层架构的高性能替代方案。

**目标：**
- 移除目前应用中的 `three_js` 方案。
- 集成 `flutter_scene` (基于Impeller) 或 `filament` 作为新的 3D 渲染核心。
- 保持现有的视觉效果与交互功能：
  - 加载和显示 GLTF 佛像。
  - 实现香柱燃烧和烟雾系统。
  - 支持手动触摸滑动视角与自动平滑绕佛。
  - 动态显示/隐藏 3D 经书。

## 2. 需求变动明细
**变更说明：**
因基于事实未能获取到闭源阶段的 `Fluorite` 提供物，转为依靠 Flutter 官方生态或其他 Filament 衍生库，确保不因为等待第三方开源而阻塞性能优化进度。这就是我们在资源受限环境下的“正确解法”。

## 3. 功能需求明细
### 3.1 3D场景初始化
- 引入新的 3D Widget 取代现有在栈底的 3D 层。
- 重建 PBR 光照与材质：重构现有的定向光，突出佛像金属立体感。

### 3.2 交互与特效
- **香的燃烧机制：** 基于替代 3D 引擎的节点变换能力处理香柱缩放和光源跟随。
- **相机控制：** 支持外部时间流驱动的参数化运动控制。

## 4. 性能预期
- **帧率优化：** 借助原生层面的渲染（如 Impeller/Vulkan），期望在老旧设备上稳定保持流畅表现，彻底消除垃圾回收(GC)和庞大的 Dart 对象树带来的偶发卡顿现象。
