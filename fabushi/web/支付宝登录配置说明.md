# 支付宝登录配置说明

## 问题分析

用户反馈支付宝登录后出现404错误，无法找到 `https://flutter.ombhrum.com/login.html` 页面。

## 根本原因

1. **重定向路径错误**：支付宝登录回调后重定向到 `/login.html`，但实际文件位于 `/assets/login.html`
2. **前端API调用错误**：支付宝登录按钮调用了错误的API接口
3. **用户信息获取不完整**：支付宝用户信息获取函数目前是模拟实现

## 已修复的问题

### 1. 重定向路径修复
- 将所有重定向URL从 `/login.html` 改为 `/assets/login.html`
- 修复了回调处理、错误处理、成功登录等所有重定向场景

### 2. 前端API调用修复
- 将支付宝登录按钮的API调用从 `/api/auth/alipay/login` 改为正确的 `/api/auth/alipay/login-url`
- 修复了请求方法从POST改为GET

### 3. 用户信息处理增强
- 改进了支付宝用户信息获取函数的配置检查
- 增加了详细的调试日志
- 完善了用户绑定和注册时的信息保存

### 4. 登录成功处理逻辑增强
- 支持已有绑定用户的直接登录成功处理
- 完善了绑定表单的提交和响应处理
- 增加了token保存和页面跳转逻辑

## 环境变量配置要求

为了使支付宝登录正常工作，需要在Cloudflare Worker中配置以下环境变量：

```
ALIPAY_APP_ID=你的支付宝应用ID
ALIPAY_PRIVATE_KEY=你的支付宝私钥
ALIPAY_PUBLIC_KEY=支付宝公钥
WORKER_URL=你的Worker域名 (如: https://flutter.ombhrum.com)
USERS_KV=你的KV命名空间绑定
```

## 支付宝登录流程

1. 用户点击支付宝登录按钮
2. 前端调用 `/api/auth/alipay/login-url` 获取授权URL
3. 用户跳转到支付宝授权页面
4. 支付宝回调到 `/api/auth/alipay/callback`
5. 系统处理回调并获取用户信息
6. 如果用户已绑定，直接登录成功
7. 如果用户未绑定，显示绑定表单
8. 用户完成绑定后登录成功

## 注意事项

1. **支付宝API集成**：目前的 `getAlipayUserInfo` 函数仍然是模拟实现，需要接入真实的支付宝API
2. **文件路径**：确保所有HTML文件都正确部署在 `/assets/` 目录下
3. **环境变量**：确保所有必要的环境变量都已正确配置
4. **域名配置**：`WORKER_URL` 必须设置为实际的Worker域名
