# D1迁移检查清单

## 📋 迁移前准备

### 环境检查
- [ ] 已安装最新版wrangler (`npm install -g wrangler@latest`)
- [ ] 已登录Cloudflare账户 (`wrangler whoami`)
- [ ] 已确认KV命名空间ID
- [ ] 已确认R2存储桶配置
- [ ] 已备份当前worker.js

### 数据备份

## 🚀 迁移执行

### 第一步：创建D1数据库
- [ ] 运行 `wrangler d1 create fabushi-db`
- [ ] 记录返回的database_id
- [ ] 更新wrangler.toml配置
- [ ] 验证配置正确性

### 第二步：初始化Schema
- [ ] 运行 `wrangler d1 execute fabushi-db --file=schema.sql --remote`
- [ ] 验证表创建成功
- [ ] 检查索引创建
- [ ] 确认表结构正确

### 第三步：数据迁移
- [ ] 备份当前worker.js
- [ ] 部署迁移脚本
- [ ] 访问 /migrate-data 端点
- [ ] 检查迁移结果
- [ ] 验证数据完整性

### 第四步：数据验证
- [ ] 用户数量一致
- [ ] 订单数量一致
- [ ] 兑换码数量一致
- [ ] 购买记录完整
- [ ] 兑换记录完整
- [ ] 邮箱映射正确

### 第五步：切换到D1
- [ ] 复制 worker-d1.js 到 worker.js
- [ ] 检查代码语法
- [ ] 部署到生产环境
- [ ] 等待部署完成
